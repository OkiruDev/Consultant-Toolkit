### Detailed Pillar Scorecard Structures for the B-BBEE Web App

Hey ScoField! Based on the Ownership scorecard example you shared (from the Excel toolkit and image), I've structured detailed breakdowns for **all main pillars** in the B-BBEE Generic Scorecard. This is designed for your web app's pillar pages or scenario tab—what-if analysis. Each pillar will have a table mimicking the format: **Indicator**, **Criteria**, **Points** (target), **%** (target or compliance level), **Prior Year Points**, **Prior Year %**, **Actual Points**, **Actual %**, **Actual [manual] Points**, **Actual [manual] %**, **Gap Points**, **Gap %**.

- **Why this format?** It matches your example exactly—targets on the left, prior/actual/manual columns for data entry/comparison, and gaps on the right. In the web app, this becomes an interactive table (e.g., using React Table or DataGrid): 
  - **Editable cells**: Prior Year and Actual [manual] for overrides/scenarios.
  - **Auto-calcs**: Actual Points/% from inputs (e.g., shareholder data for Ownership); Gaps = Target - Actual.
  - **Visuals**: Color-code gaps (red for negative, green for zero/positive); tooltips explaining criteria (e.g., "25% +1 vote means control threshold").
  - **Scenario Integration**: In what-if mode, duplicate the table, apply overrides (e.g., "Add black shareholder" recalcs Voting Rights), show deltas side-by-side.
- **Data Sources**: Based on the Okiru Excel toolkit (Revised Codes, Generic Entity). Placeholders like 0.00 for prior/actual/gaps (as in your image). At the bottom: Period selector (e.g., dropdown for year/quarter) and data date.
- **2026 Notes**: As of February 2026 (your current time in Johannesburg), the B-BBEE Codes are under draft amendments (gazetted Jan 29, 2026; comments until Mar 30, 2026). Key proposed changes: New "Transformation Fund" (20 points, 3% NPAT), ESD overhaul (emphasis on 100% Black-owned, total points up to 131-142), tighter procurement. For now, I've used the current 2019 Generic Codes (total ~120 points) matching your Excel. Add a toggle in the app: "Apply Draft 2026 Rules?" to simulate changes (e.g., add Transformation Fund row).
