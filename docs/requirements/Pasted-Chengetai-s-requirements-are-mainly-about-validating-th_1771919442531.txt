Chengetai’s requirements are mainly about **validating the scorecard system using Excel first and defining export outputs**, before moving to the full AI-driven platform. The focus is **Phase 1: Excel-driven scorecard and dashboard**, not the AI pipeline yet.

Below is a clear interpretation of what Chengetai needs and what each action item means technically.

---

## 1. Use the GP Excel as Source of Truth

### Requirement

You must use the **GP Excel toolkit** as the reference model.

Purpose:

* Verify that your system produces the same results as Excel
* Ensure formulas and calculations match exactly
* Confirm that pillar scores are correct

Meaning technically:

The system should behave like:

```
GP Excel File
     ↓
Load Data
     ↓
Apply Calculations
     ↓
Dashboard Scores
```

You must confirm:

* Ownership scores match Excel
* Management scores match Excel
* Skills scores match Excel
* Procurement scores match Excel
* Total score matches Excel

If Excel says:

```
Total Score = 68.4
```

Your system must produce:

```
Total Score = 68.4
```

This is **calculation validation**.

---

## 3. Verify Excel Input Works Correctly

### Requirement

Once Excel integration is complete:

Chengetai will verify:

* All fields load correctly
* Calculations match Excel
* Scorecards are correct

Meaning:

You must build:

```
Excel Upload
   ↓
Data Parser
   ↓
System Fields
   ↓
Score Calculation
```

Example:

Excel:

```
Black Managers = 5
Total Managers = 10
```

System must compute:

```
Management % = 50%
Score = X
```

If Excel produces different results, it fails.

### Requirement

You must build:
