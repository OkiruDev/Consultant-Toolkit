### Turning the Okiru B-BBEE Toolkit Excel into a Web App UI: Blueprint & Implementation Guide

Hey ScoField! Based on our previous chats (including the meeting action items and the Excel analysis), I understand you're focusing on building a web UI for this B-BBEE compliance toolkit. The goal is to replicate the Excel's functionality in a user-friendly web app—starting with Excel as the data source for validation, then evolving to direct inputs/uploads. This will make it interactive, with dynamic calculations, scenario planning, and exports (Excel/PPT slides).

I'll break it down by **sections** (mapping to the Excel sheets/pillars). For each, I'll explain:
- **Purpose**: What it does in the context of B-BBEE (Revised Codes of Good Practice - RCOGP).
- **Key Components**: What the UI must have (inputs, calculations, outputs).
- **How to Implement**: UI elements (e.g., forms, tables, charts), backend logic (e.g., JS formulas or server-side calcs), and ties to action items (e.g., use GP Excel for verification).
- **Dependencies**: Links to other sections or data (e.g., EAP tables affect MC/Skills).
- **Validation/Notes**: Rules, errors from Excel (e.g., grey-highlighted inputs), and testing tips.

Overall Architecture Recommendations:
- **Framework**: Use React/Vue for frontend (dynamic forms/tables/charts with libraries like Ant Design or Material-UI). Backend: Node.js/Express or Python/Flask for calcs (replicate Excel formulas in code). Database: Start with JSON/local storage; add MongoDB/Postgres for multi-tenant later (deferred per meeting).
- **Data Flow**: Upload Excel or enter data via forms → Run calcs on submit → Display results → Support scenarios (clone data, modify, compare).
- **Auth/UI Basics**: Internal use first—simple login. Responsive design for desktop/mobile. Use charts (Chart.js/Recharts) for graphs (e.g., from "Empower - Current State").
- **Testing**: Load GP Excel data (action item 15:26). Verify calcs match Excel (e.g., Deemed NPAT). Add error handling (e.g., "ID No. & Gender not matching" from MC).
- **Exports**: PDF/Excel via libraries (jsPDF, exceljs); slides as PPTX (pptxgenjs). Use templates from Chengetai (action item 18:21).
- **Scenario Planning**: Dedicated page/modal—copy base data, apply interventions (e.g., "Add black-owned supplier" updates Procurement score dynamically; action item 7:42).

Now, section-by-section:

#### 1. **Dashboard (Summary Scorecard Sheet)**
   - **Purpose**: Main overview of B-BBEE level, points, gaps. Shows current/prior/projected scores, discounted levels, YES tier. Ties into all pillars for total score (e.g., Ownership 25pts + MC 19pts + etc. = 120 max).
   - **Key Components**:
     - Inputs: Period selector (e.g., dropdown for year/quarter).
     - Calculations: Sum pillar points; apply sub-minimums (if <40% in priority pillars like Ownership/Skills/Procurement, discount level by 1); B-BBEE status (e.g., Level 1 if >=100pts).
     - Outputs: Table with pillars, targets, actuals, gaps; overall level (e.g., "Non-compliant"); charts for QoQ tracking; YES level/rewards.
   - **How to Implement**:
     - UI: Central page with tabs/cards for pillars. Use a table (e.g., Ant Table) for scorecard. Bar/pie charts for point distribution. Buttons for "Export Scorecard" (Excel) and "Generate Slides".
     - Backend: Aggregate from pillar APIs/endpoints. Formula e.g., `totalPoints = ownership + mc + skills + procurement + esd + sed; level = lookupLevel(totalPoints - discounts);`.
     - Dependencies: Pulls from "Scorecard Calculations" (gaps, tiers) and all pillars.
     - Validation/Notes: Highlight non-compliant in red (conditional formatting). Test with empty data (should show "Non-compliant"). Action item 44:53—focus here first.

#### 2. **Client Information Section**
   - **Purpose**: Entity details, financial period, revenue/NPAT/Leviable Amount. Sets baselines for all calcs (e.g., Deemed NPAT for ESD/SED targets).
   - **Key Components**:
     - Inputs: Form fields like Entity Name (text), Financial Year (date picker), Revenue/NPAT (numbers), Headcount (integer), SARS EMP201 period.
     - Calculations: None major here; feeds into Financials (e.g., NPAT margin).
     - Outputs: Display entered data; warnings like "Only input grey-highlighted cells".
   - **How to Implement**:
     - UI: Simple form page/modal. Use input fields with validation (e.g., Yup/Formik for React). Checkbox for "Combine Exec & Senior Mgmt?".
     - Backend: Store in state/session; validate numbers >=0.
     - Dependencies: Feeds Revenue/NPAT to "Financials" for averages; Headcount to MC/Skills.
     - Validation/Notes: Required fields (* marked); date formats (dd/mm/yyyy). Grey-highlight inputs as editable, others read-only.

#### 3. **Financials & Industry Norms Sections**
   - **Purpose**: Historical/projected financials (NPAT, Revenue, Leviable Amount). Norms table for Deemed NPAT (e.g., if margin <25% of norm, use indicative NPAT).
   - **Key Components**:
     - Inputs: Year-wise tables for NPAT/Revenue (multi-row form); Publication Date for norms.
     - Calculations: Averages (last 3/5 years); Deemed NPAT = MAX(current, avg); Margin = NPAT/Revenue; Compare to norm (e.g., All Industries 0.0558).
     - Outputs: Tables for historical/projected; "YES last 3 years Average".
   - **How to Implement**:
     - UI: Editable tables (e.g., React-Table with add-row). Dropdown for Industry (from norms list, e.g., "Mining 0.1625").
     - Backend: Formulas like `avgNPAT = SUM(last5Years)/5; deemedNPAT = MAX(currentNPAT, avgNPAT if margin >= norm*0.25 else indicative)`.
     - Dependencies: Norms table (lookup by industry/date); Affects ESD/SED targets (1-3% of Deemed NPAT).
     - Validation/Notes: Numbers only; auto-calc margins. Update norms manually (no live API; action item 30:46).

#### 4. **EAP (Economically Active Population) Section**
   - **Purpose**: Provincial demographics for targets (e.g., Gauteng Black Male 0.454). Affects MC/Skills gap analysis.
   - **Key Components**:
     - Inputs: Province/Year selector (dropdown, e.g., "Gauteng 2025").
     - Calculations: EAP % (e.g., Black total = AM + AF + CM + CF + IM + IF).
     - Outputs: Table with % by race/gender/disabled; National vs Provincial.
   - **How to Implement**:
     - UI: Dropdown for province/year; dynamic table showing % (e.g., DataGrid).
     - Backend: Hardcode/lookup table from Excel; `blackTotal = sum(blackGroups)`.
     - Dependencies: Used in MC Toolkit (effective targets) and Skills Toolkit (spend gaps).
     - Validation/Notes: Defaults to Gauteng (user location). Action item 30:46—expose selection in UI.

#### 5. **Ownership Pillar (Ownership Scorecard, Data, Calcs, Outstanding Debts)**
   - **Purpose**: Calculate ownership points (voting rights, economic interest, net value). Max 25pts.
   - **Key Components**:
     - Inputs: Shareholder table (name, % BO/BWO/BDG/BNE, debt, loan term, shares/value).
     - Calculations: Voting Rights % / target * points (e.g., Black % / 25% * 4); Net Value = MIN(8, (EconomicInterest * GraduationFactor)); Graduation table (1y=0.1, 9y=1).
     - Outputs: Scorecard table (criteria, target, actual, gap); Total points; Deemed Value.
   - **How to Implement**:
     - UI: Multi-row form/table for shareholders (add/delete rows). Date picker for data as at.
     - Backend: Formulas like `netValue = B / C * 8` (from Calcs sheet); Sum for totals.
     - Dependencies: Company Value Data (asset/shares); Feeds to Summary.
     - Validation/Notes: % 0-100; Debt? checkbox. Test sub-minimum (Net Value <3.2 → discount).

#### 6. **Management Control (MC) Pillar (MC Toolkit, Scorecard, Data, Calcs, Report)**
   - **Purpose**: Board/exec/employee representation. Max 19pts (e.g., Black board % / target * points).
   - **Key Components**:
     - Inputs: Employee table (name, gender, race, designation, disabled, ID, hire/term dates).
     - Calculations: % by category (e.g., Black exec / EAP target * 2pts); Gaps (actual vs optimum); Disabilities 2% target.
     - Outputs: Scorecard (board, exec, mgmt levels); Gap analysis table; Errors (ID/gender mismatch).
   - **How to Implement**:
     - UI: Searchable/editable table for employees. Dropdowns for race/gender/designation (from Validation sheet).
     - Backend: `blackExec% = countBlackExec / totalExec; points = MIN(2, blackExec% / target * 2)`.
     - Dependencies: EAP for targets; Headcount from Client Info.
     - Validation/Notes: ID validation (11-12th digits for gender); Age calc from ID. Combine Exco/Senior option.

#### 7. **Skills Development Pillar (Skills Toolkit, Scorecard, Data, Employee Demographics, Calcs)**
   - **Purpose**: Training spend on black people. Max 25pts (e.g., 3.5% Leviable on black skills).
   - **Key Components**:
     - Inputs: Training table (program, category, costs, employee ID, race/gender); Demographics (manual/auto by race/gender).
     - Calculations: Spend % / target * points (e.g., All Spend / (Leviable*0.035) * 8pts); Bursaries 2.5%; Absorption bonus.
     - Outputs: Scorecard (spend types, headcount %); Gap analysis by race/gender; Errors (no start dates).
   - **How to Implement**:
     - UI: Form for adding trainings; Breakdown charts (race/gender spend).
     - Backend: `skillsPoints = MIN(8, totalBlackSpend / (leviable * 0.035) * 8) + bonuses`.
     - Dependencies: Leviable from Financials; EAP for gaps.
     - Validation/Notes: Cap F&G/Other costs (25%/15%); Link to MC Data IDs.

#### 8. **Procurement Pillar (Procurement Scorecard, Data, PP Report, TMPS)**
   - **Purpose**: Spend on black-owned suppliers. Max 29pts (e.g., 51% BO spend / 25% target * 11pts).
   - **Key Components**:
     - Inputs: Supplier table (name, level, % BO/BWO/BDG, spend); TMPS inclusions/exclusions (cost of sales, etc.).
     - Calculations: Recognized spend (level 1=135%); TMPS = Inclusions - Exclusions; Bonus for BDG.
     - Outputs: Scorecard (categories, % spend); Report (gaps by level); Timelines (monthly spend).
   - **How to Implement**:
     - UI: Add-supplier form; Spend charts (% by level/ownership).
     - Backend: `boPoints = MIN(11, boSpend / TMPS / 0.25 * 11)`.
     - Dependencies: TMPS calcs feed totals.
     - Validation/Notes: Dilutive? flag; Level dropdown (1-9).

#### 9. **ESD & SED Pillars (ESD/SED Scorecards, Data, Calcs, Reports)**
   - **Purpose**: Contributions to enterprises/socio-econ dev (e.g., grants/loans). ESD max 17pts, SED 5pts (% of Deemed NPAT).
   - **Key Components**:
     - Inputs: Beneficiary table (name, type, amount, benefit factor, date).
     - Calculations: Recognized = Amount * Factor (e.g., Grant=1, Employee Time=0.8); SD 2% NPAT *10pts.
     - Outputs: Scorecards (target vs actual spend); Reports (initiatives, gaps).
   - **How to Implement**:
     - UI: Separate tabs for ESD/SED; Dropdown for types (from Calcs sheet).
     - Backend: `sdPoints = MIN(10, recognizedSD / (NPAT * 0.02) * 10)`.
     - Dependencies: Deemed NPAT from Financials.
     - Validation/Notes: % Black benefiting; Carry-over from prior year.

#### 10. **YES Initiative Section**
   - **Purpose**: Youth Employment Service bonuses (e.g., +1-3 levels if qualify).
   - **Key Components**:
     - Inputs: Registration date, first-time?, headcount targets (e.g., >R12M revenue → 3000 youth).
     - Calculations: Tier based on headcount/absorption; Rewards (R2k/R4k per youth).
     - Outputs: Qualification check; Tier/points.
   - **How to Implement**:
     - UI: Form with checkboxes/dropdowns; Table for options (1 vs 2).
     - Backend: If sub-minimums met and headcount >= target, `yesLevel = baseLevel - tier`.
     - Dependencies: From Summary (priority elements).
     - Validation/Notes: Non B-BBEE only.

#### 11. **Empowerment/Strategy Sections (Empower - Financials, Current State, Strategy, Actions, Risks, SED/PP/ESD/Slides, Scenario Planning)**
   - **Purpose**: Strategy pack for clients—forecasts, actions, risks, slides.
   - **Key Components**:
     - Inputs: Pillar champions, tasks/investments/timeframes, risks/clarifications.
     - Calculations: Projected scores; Rand per point (e.g., Skills R0.0k); Gaps (actual vs optimum).
     - Outputs: Graphs (projected/actual/target); Action lists; Slide previews (e.g., MC Achieved by quarter).
   - **How to Implement**:
     - UI: Dedicated strategy page with accordions (per pillar). Editable lists for actions. Scenario modal: Base vs Scenario 1 (modify inputs, recalc).
     - Backend: Clone data for scenarios; `gap = target - actual; investmentPerPoint = targetSpend / points`.
     - Dependencies: Pulls from all pillars; Graphs from Current State.
     - Validation/Notes: Manual inputs (ä/ê marked). Action item 24:44—verify with Chengetai's templates.

#### 12. **Exports & Reports**
   - **Purpose**: Generate outputs (scorecard PDF, gap analysis, strategy pack, audit pack).
   - **Key Components**: All data; Templates for slides (e.g., Skills breakdown charts).
   - **How to Implement**: Buttons on dashboard; Export full data to Excel (match sheet structure); Slides as PPTX with placeholders (e.g., insert charts).
   - Dependencies: All sections.
   - Validation/Notes: Action item 18:21—use provided templates.
